# Express Router Object

The Router object is one of Express.js's most powerful features for building scalable API applications. It enables you to modularize your routes, improve code organization, and manage complex routing requirements with ease.

## What is the Router Object?

The Express Router is a mini Express application without all the server setup but with the routing capabilities:

- It acts as a middleware and routing system
- It can have its own middleware configuration
- It can handle routes with different HTTP methods (GET, POST, etc.)
- It can be modularized into separate files
- It can be mounted to specific path prefixes in the main application

## Benefits of Using Express Router

Using the Router object offers several advantages:

1. **Code Organization**: Separate routes by functionality or resource type
2. **Modularity**: Break routes into multiple files for maintainability
3. **Reusability**: Create route modules that can be used in different parts of your application
4. **Middleware Scoping**: Apply middleware to specific groups of routes
5. **Path Prefixing**: Mount routes under specific URL prefixes
6. **Testability**: Test route handlers in isolation

## Creating a Basic Router

Here's how to create and use a basic Router object:

```javascript
// userRoutes.js
const express = require('express');
const router = express.Router();

// Define routes on the router
router.get('/', (req, res) => {
  res.json({ message: 'Get all users' });
});

router.post('/', (req, res) => {
  res.json({ message: 'Create a new user' });
});

router.get('/:id', (req, res) => {
  res.json({ message: `Get user with id ${req.params.id}` });
});

// Export the router
module.exports = router;
```

```javascript
// app.js
const express = require('express');
const userRoutes = require('./userRoutes');
const app = express();

// Mount the router on a specific path prefix
app.use('/users', userRoutes);

app.listen(3000, () => {
  console.log('Server running on port 3000');
});
```

In this example, the routes defined in `userRoutes.js` will be accessible under the `/users` prefix:

- `GET /users` - Get all users
- `POST /users` - Create a new user  
- `GET /users/123` - Get user with id 123

## Organizing Routes Across Multiple Files

For larger applications, you might want to split your routes across multiple files. Here's a recommended project structure:

```
src/
├── routes/
│   ├── index.js       # Combines all routes
│   ├── userRoutes.js  # User-related routes
│   ├── productRoutes.js  # Product-related routes
│   └── authRoutes.js  # Authentication routes
├── controllers/
│   ├── userController.js
│   ├── productController.js
│   └── authController.js
└── app.js
```

### Route Files

Each route file should focus on a specific resource:

```javascript
// routes/userRoutes.js
const express = require('express');
const router = express.Router();
const userController = require('../controllers/userController');

router.get('/', userController.getAllUsers);
router.post('/', userController.createUser);
router.get('/:id', userController.getUserById);
router.put('/:id', userController.updateUser);
router.delete('/:id', userController.deleteUser);

module.exports = router;
```

```javascript
// routes/productRoutes.js
const express = require('express');
const router = express.Router();
const productController = require('../controllers/productController');

router.get('/', productController.getAllProducts);
router.post('/', productController.createProduct);
router.get('/:id', productController.getProductById);
router.put('/:id', productController.updateProduct);
router.delete('/:id', productController.deleteProduct);

module.exports = router;
```

### Combining Routes

Use an index.js file to combine all routes:

```javascript
// routes/index.js
const express = require('express');
const userRoutes = require('./userRoutes');
const productRoutes = require('./productRoutes');
const authRoutes = require('./authRoutes');

const router = express.Router();

router.use('/users', userRoutes);
router.use('/products', productRoutes);
router.use('/auth', authRoutes);

module.exports = router;
```

```javascript
// app.js
const express = require('express');
const routes = require('./routes');
const app = express();

app.use(express.json());
app.use('/api', routes);

app.listen(3000, () => {
  console.log('Server running on port 3000');
});
```

With this structure, your routes will be accessible under paths like:
- `/api/users`
- `/api/products`
- `/api/auth`

## Router-Level Middleware

You can apply middleware specifically to a router:

```javascript
const express = require('express');
const router = express.Router();

// Router-level middleware - applies to all routes in this router
router.use((req, res, next) => {
  console.log('Router middleware:', Date.now());
  next();
});

// Apply middleware to specific routes
const validateProduct = (req, res, next) => {
  // Validation logic here
  if (!req.body.name) {
    return res.status(400).json({ error: 'Product name is required' });
  }
  next();
};

router.post('/', validateProduct, (req, res) => {
  // Create product logic
});

module.exports = router;
```

## Nesting Routers

You can nest routers to create hierarchical route structures:

```javascript
// commentRoutes.js
const express = require('express');
const router = express.Router({ mergeParams: true }); // Important for accessing parent router params

router.get('/', (req, res) => {
  // Here we can access both parent params (userId) and this router's params
  res.json({ message: `Get all comments for post ${req.params.postId}` });
});

router.post('/', (req, res) => {
  res.json({ message: `Add comment to post ${req.params.postId}` });
});

module.exports = router;
```

```javascript
// postRoutes.js
const express = require('express');
const router = express.Router();
const commentRoutes = require('./commentRoutes');

router.get('/', (req, res) => {
  res.json({ message: 'Get all posts' });
});

router.get('/:postId', (req, res) => {
  res.json({ message: `Get post with id ${req.params.postId}` });
});

// Nest the comment routes under posts
router.use('/:postId/comments', commentRoutes);

module.exports = router;
```

```javascript
// app.js
const express = require('express');
const app = express();
const postRoutes = require('./routes/postRoutes');

app.use(express.json());
app.use('/api/posts', postRoutes);

app.listen(3000, () => {
  console.log('Server running on port 3000');
});
```

This creates nested routes like:
- `GET /api/posts` - Get all posts
- `GET /api/posts/123` - Get post with id 123
- `GET /api/posts/123/comments` - Get all comments for post 123
- `POST /api/posts/123/comments` - Add comment to post 123

Note the `{ mergeParams: true }` option passed to the comment router. This ensures that the comment routes have access to parameters defined in their parent routes (like `postId`).

## Router Prefixing and Path Construction

Express Router automatically combines path segments, so you don't need to worry about handling trailing or leading slashes:

```javascript
// These are equivalent
router.use('/users/', userRouter);
router.use('/users', userRouter);

// These are also equivalent
router.get('/profile', profileHandler);
router.get('profile', profileHandler);
```

## Router Parameters

Routers can have their own parameter handlers:

```javascript
const router = express.Router();

// Parameter middleware
router.param('userId', (req, res, next, userId) => {
  // This runs before any route that includes :userId
  console.log(`Looking up user: ${userId}`);
  
  // You can fetch user data and attach to the request
  req.user = { id: userId, name: 'Example User' };
  next();
});

// Now routes with :userId parameter will have req.user available
router.get('/users/:userId', (req, res) => {
  res.json(req.user);
});
```

## Route Handlers in Separate Files

For even better organization, separate your route handlers (controllers) from your route definitions:

```javascript
// controllers/userController.js
exports.getAllUsers = (req, res) => {
  // Logic to get all users
  res.json({ message: 'Getting all users' });
};

exports.getUserById = (req, res) => {
  // Logic to get specific user
  res.json({ message: `Getting user ${req.params.id}` });
};

exports.createUser = (req, res) => {
  // Logic to create user
  res.json({ message: 'Creating new user' });
};
```

```javascript
// routes/userRoutes.js
const express = require('express');
const router = express.Router();
const { getAllUsers, getUserById, createUser } = require('../controllers/userController');

router.get('/', getAllUsers);
router.get('/:id', getUserById);
router.post('/', createUser);

module.exports = router;
```

## Best Practices for Route Organization

1. **Group by Resource**: Organize routes by the resource they operate on (users, products, etc.)

2. **Use Consistent Naming Conventions**:
   - Use plural nouns for resources (users, products)
   - Use kebab-case for multi-word resources (order-items)
   - Use descriptive action names when RESTful isn't sufficient (auth/login, users/reset-password)

3. **Separate Route Definition from Business Logic**:
   - Routes should handle HTTP concerns (request parsing, response formatting)
   - Controllers should contain business logic
   - Models should handle data access and validation

4. **Apply the Right Level of Nesting**:
   - Avoid deep nesting (no more than 2-3 levels)
   - Consider using query parameters instead of deep nesting
   - Use consistent patterns for related resources

5. **Use Middleware Effectively**:
   - Authentication/authorization at router level
   - Validation middleware for specific routes
   - Resource loading middleware with router.param()

6. **Handle API Versioning**:
   ```javascript
   // app.js
   const express = require('express');
   const v1Routes = require('./routes/v1');
   const v2Routes = require('./routes/v2');
   const app = express();
   
   app.use('/api/v1', v1Routes);
   app.use('/api/v2', v2Routes);
   ```

7. **Document Routes**:
   - Include JSDoc comments for each route
   - Consider using tools like Swagger/OpenAPI to generate documentation
   - Document expected request/response formats

8. **Handle Base Route Documentation**:
   ```javascript
   router.get('/', (req, res) => {
     res.json({
       availableEndpoints: {
         users: '/api/users',
         products: '/api/products',
         // etc.
       }
     });
   });
   ```

## Advanced Router Techniques

### Route Method Chaining

You can chain HTTP methods for the same path:

```javascript
router.route('/users/:id')
  .get((req, res) => {
    // Get user
    res.json({ message: `Get user ${req.params.id}` });
  })
  .put((req, res) => {
    // Update user
    res.json({ message: `Update user ${req.params.id}` });
  })
  .delete((req, res) => {
    // Delete user
    res.json({ message: `Delete user ${req.params.id}` });
  });
```

### Dynamic Route Loading

For very large applications, you can dynamically load route files:

```javascript
// routes/index.js
const express = require('express');
const fs = require('fs');
const path = require('path');
const router = express.Router();

// Dynamically load all routes
fs.readdirSync(__dirname)
  .filter(file => file !== 'index.js' && file.endsWith('.js'))
  .forEach(file => {
    const routeName = file.split('.')[0];
    const route = require(`./${file}`);
    router.use(`/${routeName.replace('Routes', '')}`, route);
  });

module.exports = router;
```

## Debugging Router Issues

When debugging router issues:

1. **Check Mount Points**: Ensure routers are mounted at the correct paths
2. **Verify Parameter Access**: When nesting routers, confirm you're using `mergeParams: true` if needed
3. **Monitor Router Order**: Routers follow the same middleware order rules as Express - first declared, first executed
4. **Inspect Path Construction**: Check that the combined paths match what you expect
