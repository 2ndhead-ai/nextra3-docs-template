# Integration Testing in Express.js APIs

Integration testing is a critical phase in testing Express.js applications where we verify that different parts of the application work together as expected. Unlike unit tests that focus on isolated components, integration tests examine how multiple components interact with each other in real-world scenarios.

## What is Integration Testing for APIs?

Integration testing for APIs focuses on testing the complete request-response cycle, including:

- HTTP request handling
- Route matching and parameter extraction
- Middleware execution
- Controller logic
- Database operations
- Response generation and formatting

The goal is to verify that all these components work together correctly to deliver the expected API behavior, ensuring that your endpoints respond appropriately to different requests and handle various scenarios properly.

## Setting Up Testing Tools

### Supertest

Supertest is the most popular library for testing Express.js APIs, allowing you to send HTTP requests to your Express application and make assertions about the responses.

First, install the necessary packages:

```
npm install --save-dev supertest jest
```

Basic setup with Jest and Supertest:

```javascript
const request = require('supertest');
const app = require('../app'); // Your Express app

describe('User API', () => {
  test('GET /api/users should return all users', async () => {
    const response = await request(app)
      .get('/api/users')
      .expect('Content-Type', /json/)
      .expect(200);
      
    expect(response.body).toHaveProperty('users');
    expect(Array.isArray(response.body.users)).toBeTruthy();
  });
});
```

### Mocha and Chai HTTP

Alternatively, you can use Mocha with Chai HTTP:

```
npm install --save-dev mocha chai chai-http
```

Basic setup with Mocha and Chai HTTP:

```javascript
const chai = require('chai');
const chaiHttp = require('chai-http');
const app = require('../app');
const expect = chai.expect;

chai.use(chaiHttp);

describe('User API', () => {
  it('should GET all users', (done) => {
    chai.request(app)
      .get('/api/users')
      .end((err, res) => {
        expect(res).to.have.status(200);
        expect(res.body).to.be.an('object');
        expect(res.body).to.have.property('users');
        expect(res.body.users).to.be.an('array');
        done();
      });
  });
});
```

## Testing Full Request-Response Cycles

### Testing GET Endpoints

```javascript
// Using Jest and Supertest
describe('GET /api/products', () => {
  test('should return all products', async () => {
    const response = await request(app)
      .get('/api/products')
      .expect(200);
      
    expect(response.body.data.length).toBeGreaterThan(0);
    expect(response.body.success).toBe(true);
  });
  
  test('should return products filtered by category', async () => {
    const category = 'electronics';
    const response = await request(app)
      .get(`/api/products?category=${category}`)
      .expect(200);
      
    response.body.data.forEach(product => {
      expect(product.category).toBe(category);
    });
  });
});
```

### Testing POST Endpoints

```javascript
describe('POST /api/products', () => {
  test('should create a new product', async () => {
    const newProduct = {
      name: 'Test Product',
      price: 99.99,
      category: 'test',
      description: 'A test product'
    };
    
    const response = await request(app)
      .post('/api/products')
      .send(newProduct)
      .expect('Content-Type', /json/)
      .expect(201);
      
    expect(response.body.data).toHaveProperty('id');
    expect(response.body.data.name).toBe(newProduct.name);
  });
  
  test('should return 400 when required fields are missing', async () => {
    const invalidProduct = {
      name: 'Invalid Product'
      // price is missing
    };
    
    const response = await request(app)
      .post('/api/products')
      .send(invalidProduct)
      .expect(400);
      
    expect(response.body).toHaveProperty('error');
  });
});
```

### Testing PUT and DELETE Endpoints

```javascript
describe('PUT /api/products/:id', () => {
  test('should update an existing product', async () => {
    const productId = 1;
    const updates = {
      price: 129.99,
      description: 'Updated description'
    };
    
    const response = await request(app)
      .put(`/api/products/${productId}`)
      .send(updates)
      .expect(200);
      
    expect(response.body.data.price).toBe(updates.price);
    expect(response.body.data.description).toBe(updates.description);
  });
});

describe('DELETE /api/products/:id', () => {
  test('should delete a product', async () => {
    const productId = 1;
    
    await request(app)
      .delete(`/api/products/${productId}`)
      .expect(204);
      
    // Verify product is deleted
    await request(app)
      .get(`/api/products/${productId}`)
      .expect(404);
  });
});
```

## Testing with Real vs. Mock Databases

### Using a Test Database

For integration tests with a real database:

```javascript
// test-setup.js
const mongoose = require('mongoose');
require('dotenv').config({ path: '.env.test' });

beforeAll(async () => {
  await mongoose.connect(process.env.TEST_MONGODB_URI, {
    useNewUrlParser: true,
    useUnifiedTopology: true
  });
});

afterAll(async () => {
  await mongoose.connection.dropDatabase();
  await mongoose.connection.close();
});

beforeEach(async () => {
  // Clear collections before each test
  const collections = mongoose.connection.collections;
  for (const key in collections) {
    await collections[key].deleteMany({});
  }
});
```

### Using In-Memory MongoDB

MongoDB Memory Server provides an in-memory MongoDB server for testing:

```
npm install --save-dev mongodb-memory-server
```

```javascript
// test-setup.js
const { MongoMemoryServer } = require('mongodb-memory-server');
const mongoose = require('mongoose');

let mongoServer;

beforeAll(async () => {
  mongoServer = await MongoMemoryServer.create();
  const uri = mongoServer.getUri();
  await mongoose.connect(uri);
});

afterAll(async () => {
  await mongoose.disconnect();
  await mongoServer.stop();
});

beforeEach(async () => {
  const collections = mongoose.connection.collections;
  for (const key in collections) {
    await collections[key].deleteMany({});
  }
});
```

### Using SQLite for SQL Tests

For SQL databases, use SQLite in memory:

```javascript
// test-db-setup.js
const { Sequelize } = require('sequelize');

let sequelize;

beforeAll(async () => {
  sequelize = new Sequelize('sqlite::memory:', {
    logging: false
  });
  
  // Define models
  const User = sequelize.define('User', {
    // model definition
  });
  
  // Sync all models
  await sequelize.sync({ force: true });
});

afterAll(async () => {
  await sequelize.close();
});

beforeEach(async () => {
  // Clear data before each test
  await sequelize.sync({ force: true });
});
```

### Mocking Database Operations

For faster tests, you can mock database operations:

```javascript
// Mock MongoDB models
jest.mock('../models/user.model', () => ({
  find: jest.fn(),
  findById: jest.fn(),
  create: jest.fn(),
  findByIdAndUpdate: jest.fn(),
  findByIdAndDelete: jest.fn()
}));

const User = require('../models/user.model');

beforeEach(() => {
  jest.clearAllMocks();
});

test('GET /api/users should return all users', async () => {
  const mockUsers = [
    { id: '1', name: 'User 1' },
    { id: '2', name: 'User 2' }
  ];
  
  User.find.mockResolvedValue(mockUsers);
  
  const response = await request(app)
    .get('/api/users')
    .expect(200);
    
  expect(response.body).toHaveProperty('users');
  expect(response.body.users).toEqual(mockUsers);
});
```

## Testing Authentication in Integration Tests

### Authenticated Requests with JWT

```javascript
const jwt = require('jsonwebtoken');
const { JWT_SECRET } = process.env;

// Helper function to generate valid JWT token
function generateAuthToken(user) {
  return jwt.sign({ id: user.id, role: user.role }, JWT_SECRET, { expiresIn: '1h' });
}

describe('Protected API Routes', () => {
  test('should access protected route with valid token', async () => {
    // Generate token for test user
    const testUser = { id: 123, role: 'admin' };
    const token = generateAuthToken(testUser);
    
    const response = await request(app)
      .get('/api/admin/dashboard')
      .set('Authorization', `Bearer ${token}`)
      .expect(200);
      
    expect(response.body).toHaveProperty('data');
  });
  
  test('should reject access with invalid token', async () => {
    await request(app)
      .get('/api/admin/dashboard')
      .set('Authorization', 'Bearer invalid-token')
      .expect(401);
  });
  
  test('should reject access with missing token', async () => {
    await request(app)
      .get('/api/admin/dashboard')
      .expect(401);
  });
});
```

### Testing with Session Authentication

```javascript
const agent = request.agent(app);

describe('Session-based Auth', () => {
  beforeEach(async () => {
    // Log in before each test
    await agent
      .post('/api/login')
      .send({ username: 'testuser', password: 'password' });
  });
  
  test('should access protected route after login', async () => {
    const response = await agent
      .get('/api/profile')
      .expect(200);
      
    expect(response.body).toHaveProperty('username', 'testuser');
  });
});
```

### Testing Role-Based Access Control

```javascript
describe('Role-based Access Control', () => {
  test('admin can access admin routes', async () => {
    const adminToken = generateAuthToken({ id: 1, role: 'admin' });
    
    await request(app)
      .get('/api/admin/users')
      .set('Authorization', `Bearer ${adminToken}`)
      .expect(200);
  });
  
  test('regular user cannot access admin routes', async () => {
    const userToken = generateAuthToken({ id: 2, role: 'user' });
    
    await request(app)
      .get('/api/admin/users')
      .set('Authorization', `Bearer ${userToken}`)
      .expect(403);
  });
});
```

## Best Practices for Integration Testing

### 1. Isolate Tests from External Services

```javascript
// Mock external API calls
jest.mock('axios');
const axios = require('axios');

test('should handle external API errors', async () => {
  // Mock a failed API call
  axios.get.mockRejectedValue(new Error('API timeout'));
  
  const response = await request(app)
    .get('/api/external-data')
    .expect(500);
    
  expect(response.body.error).toBeTruthy();
});
```

### 2. Create Test Data Helpers

```javascript
// test-utils.js
async function createTestUser() {
  return User.create({
    username: 'testuser',
    email: 'test@example.com',
    password: 'password123'
  });
}

// In tests
const user = await createTestUser();
const token = generateAuthToken(user);
```

### 3. Test Different Response Formats

```javascript
test('should return JSON or XML based on Accept header', async () => {
  // Test JSON response
  const jsonResponse = await request(app)
    .get('/api/users/1')
    .set('Accept', 'application/json')
    .expect('Content-Type', /json/)
    .expect(200);
    
  // Test XML response
  const xmlResponse = await request(app)
    .get('/api/users/1')
    .set('Accept', 'application/xml')
    .expect('Content-Type', /xml/)
    .expect(200);
});
```

### 4. Test Pagination and Filtering

```javascript
test('should paginate results correctly', async () => {
  const page1 = await request(app)
    .get('/api/products?page=1&limit=10')
    .expect(200);
    
  const page2 = await request(app)
    .get('/api/products?page=2&limit=10')
    .expect(200);
    
  expect(page1.body.data).not.toEqual(page2.body.data);
  expect(page1.body.pagination.next.page).toBe(2);
});
```

### 5. Test Error Handling

```javascript
test('should handle database connection errors gracefully', async () => {
  // Force database error
  const originalFind = Product.find;
  Product.find = jest.fn().mockRejectedValue(new Error('DB connection lost'));
  
  const response = await request(app)
    .get('/api/products')
    .expect(500);
    
  expect(response.body).toHaveProperty('error');
  
  // Restore original method
  Product.find = originalFind;
});
```

## Conclusion

Integration testing is vital for ensuring your Express.js API works correctly as a whole. By thoroughly testing the request-response cycle, database interactions, and authentication flows, you can catch issues that might be missed by unit tests alone.

Key takeaways:
- Use Supertest or similar tools to simulate HTTP requests
- Test with real databases for complete verification or in-memory databases for speed
- Properly test authentication and authorization flows
- Verify error handling and edge cases
- Ensure your tests are isolated and repeatable

By implementing comprehensive integration tests alongside your unit tests, you'll build more robust, reliable Express.js APIs that work correctly in real-world scenarios.
