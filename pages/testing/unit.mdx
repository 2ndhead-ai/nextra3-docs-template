# Unit Testing in Express.js

Unit testing is a critical practice in building reliable Express.js applications. This guide covers everything you need to know about effectively testing your Express.js code at the component level.

## What is Unit Testing?

Unit testing is the practice of testing individual components of your application in isolation to ensure they behave as expected. In Express.js applications, units typically include:

- Individual route handlers
- Middleware functions
- Controller methods
- Utility functions
- Data models and services

The key characteristic of unit tests is that they focus on testing a single "unit" of functionality without relying on external dependencies like databases, file systems, or APIs.

## Why Unit Testing Matters for Express.js

Unit testing is particularly valuable for Express.js applications because:

1. **API Reliability**: Ensures each endpoint behaves correctly under various conditions
2. **Middleware Verification**: Confirms middleware functions correctly process requests and responses
3. **Refactoring Safety**: Allows you to improve code without breaking functionality
4. **Documentation**: Provides examples of how components should work
5. **Component Isolation**: Helps identify exactly where issues occur
6. **Development Speed**: Catches bugs earlier in the development cycle
7. **Maintainability**: Makes code easier to understand and modify

## Setting Up Testing Frameworks

### Setting Up Jest

Jest is a popular, zero-configuration testing framework that works well with Express.js:

1. Install Jest:
   ```
   npm install --save-dev jest @types/jest supertest
   ```

2. Add test scripts to your package.json:
   ```json
   "scripts": {
     "test": "jest",
     "test:watch": "jest --watch",
     "test:coverage": "jest --coverage"
   }
   ```

3. Create a Jest configuration file (jest.config.js):
   ```javascript
   module.exports = {
     testEnvironment: 'node',
     testMatch: ['**/__tests__/**/*.js', '**/?(*.)+(spec|test).js'],
     collectCoverageFrom: ['src/**/*.js'],
     coveragePathIgnorePatterns: ['/node_modules/', '/tests/'],
     verbose: true
   };
   ```

### Setting Up Mocha with Chai

Mocha is a flexible testing framework that pairs well with the Chai assertion library:

1. Install Mocha, Chai, and related utilities:
   ```
   npm install --save-dev mocha chai sinon supertest
   ```

2. Add test scripts to your package.json:
   ```json
   "scripts": {
     "test": "mocha 'tests/**/*.js'",
     "test:watch": "mocha --watch 'tests/**/*.js'"
   }
   ```

3. Create a test helper file (test/setup.js):
   ```javascript
   const chai = require('chai');
   global.expect = chai.expect;
   global.sinon = require('sinon');
   ```

## Testing Express.js Route Handlers

Route handlers are functions that process HTTP requests and send responses. Here's how to test them effectively:

### Approach 1: Direct Function Testing

Test the route handler function directly by providing mock request and response objects:

```javascript
// users.controller.js
exports.getUser = async (req, res) => {
  try {
    const user = await User.findById(req.params.id);
    if (!user) {
      return res.status(404).json({ message: 'User not found' });
    }
    res.status(200).json(user);
  } catch (error) {
    res.status(500).json({ message: 'Server error' });
  }
};

// users.controller.test.js
const { getUser } = require('../controllers/users.controller');
const User = require('../models/user.model');

// Using Jest
jest.mock('../models/user.model');

describe('User Controller', () => {
  let req, res;
  
  beforeEach(() => {
    req = {
      params: { id: 'user123' }
    };
    res = {
      status: jest.fn().mockReturnThis(),
      json: jest.fn()
    };
  });

  test('getUser returns user when found', async () => {
    const mockUser = { id: 'user123', name: 'John Doe' };
    User.findById.mockResolvedValue(mockUser);
    
    await getUser(req, res);
    
    expect(res.status).toHaveBeenCalledWith(200);
    expect(res.json).toHaveBeenCalledWith(mockUser);
  });

  test('getUser returns 404 when user not found', async () => {
    User.findById.mockResolvedValue(null);
    
    await getUser(req, res);
    
    expect(res.status).toHaveBeenCalledWith(404);
    expect(res.json).toHaveBeenCalledWith({ message: 'User not found' });
  });

  test('getUser handles exceptions', async () => {
    User.findById.mockRejectedValue(new Error('Database error'));
    
    await getUser(req, res);
    
    expect(res.status).toHaveBeenCalledWith(500);
    expect(res.json).toHaveBeenCalledWith({ message: 'Server error' });
  });
});
```

### Approach 2: Request Simulation with Supertest

Test the route by making simulated HTTP requests to your Express app:

```javascript
// app.js
const express = require('express');
const userRoutes = require('./routes/user.routes');
const app = express();
app.use(express.json());
app.use('/api/users', userRoutes);
module.exports = app;

// users.test.js
const request = require('supertest');
const app = require('../app');
const User = require('../models/user.model');

// Using Jest
jest.mock('../models/user.model');

describe('User API', () => {
  test('GET /api/users/:id returns user when found', async () => {
    const mockUser = { id: 'user123', name: 'John Doe' };
    User.findById.mockResolvedValue(mockUser);
    
    const response = await request(app)
      .get('/api/users/user123')
      .expect('Content-Type', /json/)
      .expect(200);
      
    expect(response.body).toEqual(mockUser);
  });

  test('GET /api/users/:id returns 404 when user not found', async () => {
    User.findById.mockResolvedValue(null);
    
    await request(app)
      .get('/api/users/invalid')
      .expect('Content-Type', /json/)
      .expect(404);
  });
});
```

## Testing Middleware Functions

Middleware functions are essential components in Express.js applications that process requests before they reach route handlers:

```javascript
// auth.middleware.js
const jwt = require('jsonwebtoken');

const authMiddleware = (req, res, next) => {
  const token = req.header('x-auth-token');
  
  if (!token) {
    return res.status(401).json({ message: 'No token, authorization denied' });
  }
  
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded.user;
    next();
  } catch (err) {
    res.status(401).json({ message: 'Token is not valid' });
  }
};

module.exports = authMiddleware;

// auth.middleware.test.js
const jwt = require('jsonwebtoken');
const authMiddleware = require('../middleware/auth.middleware');

// Using Jest
jest.mock('jsonwebtoken');

describe('Auth Middleware', () => {
  let req, res, next;
  
  beforeEach(() => {
    req = {
      header: jest.fn()
    };
    res = {
      status: jest.fn().mockReturnThis(),
      json: jest.fn()
    };
    next = jest.fn();
  });
  
  test('should call next() when token is valid', () => {
    const mockDecodedToken = { user: { id: 1, name: 'Test User' } };
    req.header.mockReturnValue('valid-token');
    jwt.verify.mockReturnValue(mockDecodedToken);
    
    authMiddleware(req, res, next);
    
    expect(req.user).toEqual(mockDecodedToken.user);
    expect(next).toHaveBeenCalled();
    expect(res.status).not.toHaveBeenCalled();
  });
  
  test('should return 401 when no token is provided', () => {
    req.header.mockReturnValue(null);
    
    authMiddleware(req, res, next);
    
    expect(res.status).toHaveBeenCalledWith(401);
    expect(res.json).toHaveBeenCalledWith({ message: 'No token, authorization denied' });
    expect(next).not.toHaveBeenCalled();
  });
  
  test('should return 401 when token is invalid', () => {
    req.header.mockReturnValue('invalid-token');
    jwt.verify.mockImplementation(() => {
      throw new Error('Invalid token');
    });
    
    authMiddleware(req, res, next);
    
    expect(res.status).toHaveBeenCalledWith(401);
    expect(res.json).toHaveBeenCalledWith({ message: 'Token is not valid' });
    expect(next).not.toHaveBeenCalled();
  });
});
```

## Testing Controllers

Controllers hold the business logic of your application. Here's how to test them effectively:

```javascript
// product.controller.js
const Product = require('../models/product.model');

exports.createProduct = async (req, res) => {
  try {
    const { name, price, description } = req.body;
    
    // Validation
    if (!name || !price) {
      return res.status(400).json({ message: 'Name and price are required' });
    }
    
    const newProduct = new Product({
      name,
      price,
      description,
      createdBy: req.user.id
    });
    
    await newProduct.save();
    
    res.status(201).json(newProduct);
  } catch (error) {
    console.error(error);
    res.status(500).json({ message: 'Server error' });
  }
};

// product.controller.test.js
const { createProduct } = require('../controllers/product.controller');
const Product = require('../models/product.model');

// Using Jest
jest.mock('../models/product.model');

describe('Product Controller', () => {
  let req, res;
  
  beforeEach(() => {
    req = {
      body: {
        name: 'Test Product',
        price: 99.99,
        description: 'A test product'
      },
      user: { id: 'user123' }
    };
    
    res = {
      status: jest.fn().mockReturnThis(),
      json: jest.fn()
    };
    
    // Mock the Product model constructor and its save method
    Product.mockImplementation(function(data) {
      this.data = data;
      this.save = jest.fn().mockResolvedValue(this);
    });
  });
  
  test('createProduct should create a new product', async () => {
    await createProduct(req, res);
    
    expect(res.status).toHaveBeenCalledWith(201);
    expect(res.json).toHaveBeenCalled();
    
    const responseProduct = res.json.mock.calls[0][0];
    expect(responseProduct.data.name).toBe('Test Product');
    expect(responseProduct.data.price).toBe(99.99);
    expect(responseProduct.data.createdBy).toBe('user123');
  });
  
  test('createProduct should return 400 if name is missing', async () => {
    req.body.name = undefined;
    
    await createProduct(req, res);
    
    expect(res.status).toHaveBeenCalledWith(400);
    expect(res.json).toHaveBeenCalledWith({ message: 'Name and price are required' });
  });
  
  test('createProduct should handle errors', async () => {
    Product.mockImplementation(() => {
      return {
        save: jest.fn().mockRejectedValue(new Error('Database error'))
      };
    });
    
    await createProduct(req, res);
    
    expect(res.status).toHaveBeenCalledWith(500);
    expect(res.json).toHaveBeenCalledWith({ message: 'Server error' });
  });
});
```

## Mocking Dependencies

Effective unit testing requires isolating the code under test by mocking external dependencies:

### Mocking Databases

```javascript
// Using Jest to mock Mongoose
jest.mock('../models/user.model', () => ({
  findById: jest.fn(),
  findOne: jest.fn(),
  create: jest.fn(),
  findByIdAndUpdate: jest.fn()
}));

// Alternatively, using a mock MongoDB memory server for more realistic testing
const { MongoMemoryServer } = require('mongodb-memory-server');
const mongoose = require('mongoose');

let mongoServer;

beforeAll(async () => {
  mongoServer = await MongoMemoryServer.create();
  await mongoose.connect(mongoServer.getUri(), { useNewUrlParser: true });
});

afterAll(async () => {
  await mongoose.disconnect();
  await mongoServer.stop();
});
```

### Mocking External Services

```javascript
// payment.service.js
const axios = require('axios');

exports.processPayment = async (amount, cardDetails) => {
  try {
    const response = await axios.post('https://payment-gateway.com/api/process', {
      amount,
      ...cardDetails
    });
    return response.data;
  } catch (error) {
    throw new Error('Payment processing failed');
  }
};

// payment.service.test.js
const axios = require('axios');
const paymentService = require('../services/payment.service');

jest.mock('axios');

describe('Payment Service', () => {
  test('processPayment should complete successfully', async () => {
    const mockResponse = { data: { success: true, transactionId: '123456' } };
    axios.post.mockResolvedValue(mockResponse);
    
    const result = await paymentService.processPayment(100, { cardNumber: '4111111111111111' });
    
    expect(result).toEqual(mockResponse.data);
    expect(axios.post).toHaveBeenCalledWith(
      'https://payment-gateway.com/api/process',
      expect.objectContaining({ amount: 100 })
    );
  });
  
  test('processPayment should handle errors', async () => {
    axios.post.mockRejectedValue(new Error('API error'));
    
    await expect(paymentService.processPayment(100, { cardNumber: '4111111111111111' }))
      .rejects
      .toThrow('Payment processing failed');
  });
});
```

## Best Practices for Effective Unit Tests

### 1. Follow the AAA Pattern (Arrange-Act-Assert)

Structure your tests for clarity:

```javascript
test('should calculate total price correctly', () => {
  // Arrange
  const cart = [
    { name: 'Product A', price: 10, quantity: 2 },
    { name: 'Product B', price: 15, quantity: 1 }
  ];
  
  // Act
  const total = calculateTotal(cart);
  
  // Assert
  expect(total).toBe(35);
});
```

### 2. Test One Thing Per Test

Keep tests focused on a single behavior:

```javascript
// Bad: Testing multiple behaviors
test('auth middleware functionality', () => {
  const req = { header: () => null };
  authMiddleware(req, res, next);
  expect(res.status).toHaveBeenCalledWith(401); // No token test
  
  req.header = () => 'invalid';
  authMiddleware(req, res, next);
  expect(res.status).toHaveBeenCalledWith(401); // Invalid token test
});

// Good: Testing one behavior per test
test('auth middleware returns 401 when no token provided', () => {
  const req = { header: () => null };
  authMiddleware(req, res, next);
  expect(res.status).toHaveBeenCalledWith(401);
});

test('auth middleware returns 401 when token is invalid', () => {
  const req = { header: () => 'invalid' };
  jwt.verify.mockImplementation(() => { throw new Error(); });
  authMiddleware(req, res, next);
  expect(res.status).toHaveBeenCalledWith(401);
});
```

### 3. Use Descriptive Test Names

Name your tests to clearly describe the expected behavior:

```javascript
// Unclear
test('getUser test', () => {});

// Clear and descriptive
test('getUser returns 404 status when user ID does not exist', () => {});
```

### 4. Test Edge Cases and Error Scenarios

Don't just test the happy path; test boundary conditions and error handling:

```javascript
describe('userValidator middleware', () => {
  test('passes valid user data', () => {});
  test('rejects when name is missing', () => {});
  test('rejects when email format is invalid', () => {});
  test('rejects when password is less than 8 characters', () => {});
  test('trims whitespace from name field', () => {});
});
```

### 5. Keep Tests Fast and Independent

Tests should be quick and not rely on other tests:

```javascript
// Before each test, reset the state
beforeEach(() => {
  jest.clearAllMocks();
  req = { body: {}, params: {} };
  res = {
    status: jest.fn().mockReturnThis(),
    json: jest.fn()
  };
});
```

### 6. Use Test Coverage Tools

Monitor and improve your test coverage:

```
npm test -- --coverage
```

Aim for high coverage while focusing on critical paths and complex logic.

### 7. Avoid Testing Implementation Details

Test behavior rather than internal implementation:

```javascript
// Too implementation-focused
test('calls database with correct query', () => {
  const spy = jest.spyOn(db, 'query');
  userService.findByEmail('test@example.com');
  expect(spy).toHaveBeenCalledWith('SELECT * FROM users WHERE email = ?', ['test@example.com']);
});

// Better - focuses on behavior
test('returns user when email exists', async () => {
  const mockUser = { id: 1, email: 'test@example.com' };
  jest.spyOn(db, 'query').mockResolvedValue([mockUser]);
  
  const result = await userService.findByEmail('test@example.com');
  
  expect(result).toEqual(mockUser);
});
```

## Handling Asynchronous Code

Express.js applications often involve asynchronous operations. Here's how to test them properly:

### Using Async/Await

```javascript
test('getUserById returns user when found', async () => {
  User.findById.mockResolvedValue({ id: '123', name: 'Test User' });
  
  await userController.getUserById(req, res);
  
  expect(res.json).toHaveBeenCalledWith(expect.objectContaining({ name: 'Test User' }));
});
```

### Testing Error Handling

```javascript
test('getUserById handles database errors properly', async () => {
  const errorMessage = 'Database connection failed';
  User.findById.mockRejectedValue(new Error(errorMessage));
  
  await userController.getUserById(req, res);
  
  expect(res.status).toHaveBeenCalledWith(500);
  expect(res.json).toHaveBeenCalledWith(expect.objectContaining({
    error: expect.any(String)
  }));
});
```

## Conclusion

Unit testing is an essential practice for building reliable Express.js applications. By testing individual components in isolation with mocked dependencies, you can catch bugs early and ensure your API behaves as expected.

Remember these key points:
- Focus on testing behavior, not implementation details
- Mock external dependencies to keep tests fast and isolated
- Test both success and error scenarios
- Keep tests independent and maintainable
- Use appropriate testing frameworks and tools for your needs

With consistent unit testing practices, you'll build more robust Express.js applications that are easier to maintain and extend over time.
